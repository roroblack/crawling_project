## 2. 파싱 알고리즘

### 핵심 아이디어

A열·B열의 병합 셀을 **Forward-fill**(이전 값 유지)로 처리한 뒤,  
각 연료 섹션의 **`소계/계` 행 1개만 직접 읽어** 지역별 값을 취득한다.

- `B열 == '소계'` (forward-fill 후) : 모든 차종(승용·승합·화물·특수)의 합  
- `C열 == '계'` : 비사업용 + 사업용의 합  
- 즉, **`소계/계` 행 = 해당 연료의 전체 지역별 누적 등록수** (정부 공식 집계값)

### 수소차 최종 합산

`TARGET_FUELS = {"수소", "수소전기"}` 두 연료 각각의 `소계/계` 행을 합산한다.

```
수소차 등록 총수 = 수소(소계/계) + 수소전기(소계/계)
```

### 코드 흐름 (핵심 부분)

```python
current_fuel    = ""
current_vehicle = ""

for row in all_rows[header_row_idx + 1:]:
    # A열 forward-fill
    fuel_raw = str(row[0]).strip() if row[0] else ""
    if fuel_raw:
        current_fuel    = fuel_raw
        current_vehicle = ""      # 연료 섹션 전환 시 차종 초기화
    fuel = current_fuel

    if fuel_filter and fuel not in fuel_filter:
        continue                  # 수소·수소전기 외 연료 건너뜀

    # B열 forward-fill
    vehicle_raw = str(row[1]).strip() if row[1] else ""
    if vehicle_raw:
        current_vehicle = vehicle_raw
    vehicle = current_vehicle

    usage = str(row[2]).strip() if row[2] else ""

    # 소계/계 행만 처리
    if vehicle != "소계" or usage != "계":
        continue

    for region, col_idx in region_col_map.items():
        raw   = row[col_idx]
        count = int(raw) if isinstance(raw, (int, float)) and raw == raw else 0
        region_totals[region] += count
```

## 6. 구버전(v1) 파싱 코드 — 개별 행 합산 방식

> 레거시 보관 위치: `legacy/crawler_molit_v1_individual_sum.py`  
> 현재 운영 코드: `crawler_molit.py` (v2, 소계/계 직접 읽기)

### v1 핵심 로직 (`_parse` 내부)

```python
# 데이터 행을 순회하며 지정된 연료 행의 값을 지역별로 합산한다.
# ─ 병합 셀 대응: A열(연료) · B열(차종)이 None이면 이전 값을 유지한다.
#   2018-2022 파일은 같은 연료/차종 그룹의 행을 병합 셀로 표현하므로
#   첫 행을 제외한 나머지 행은 A·B가 None 으로 읽힌다.
current_fuel    = ""
current_vehicle = ""

for row in all_rows[header_row_idx + 1:]:
    if not row:
        continue

    # A열: 연료 — 새 값이 있으면 갱신 및 차종 초기화
    fuel_raw = str(row[0]).strip() if row[0] else ""
    if fuel_raw:
        current_fuel    = fuel_raw
        current_vehicle = ""           # 연료 섹션이 바뀌면 차종 초기화
    fuel = current_fuel

    if not fuel:
        continue                       # 아직 연료가 확정되지 않은 초기 행

    if fuel_filter and fuel not in fuel_filter:
        continue                       # 대상 외 연료는 건너뜀

    # B열: 차종 — 새 값이 있으면 갱신 (병합 셀이면 이전 차종 유지)
    vehicle_raw = str(row[1]).strip() if row[1] else ""
    if vehicle_raw:
        current_vehicle = vehicle_raw
    vehicle = current_vehicle

    usage = str(row[2]).strip() if row[2] else ""     # C열: 용도

    # 소계(B='소계') · 합계(C='계') 행은 중복 집계이므로 제외
    if not vehicle or not usage or vehicle == "소계" or usage == "계":
        continue

    for region, col_idx in region_col_map.items():
        if col_idx >= len(row):
            continue
        raw = row[col_idx]
        count = int(raw) if isinstance(raw, (int, float)) and raw == raw else 0
        region_totals[region] = region_totals.get(region, 0) + count
```

### v1 vs v2 비교

| 항목 | v1 (개별 행 합산) | v2 (소계/계 직접 읽기) |
|:---|:---|:---|
| 읽는 행 수 | 연료별 8행 (차종×용도 조합) | 연료별 **1행** (소계/계) |
| 제외 조건 | `vehicle == "소계" or usage == "계"` | `vehicle != "소계" or usage != "계"` |
| 결과값 | 수학적으로 동일 | 수학적으로 동일 |
| 공식 집계 활용 | 간접 (재합산) | 직접 (정부 공식값) |
| 코드 직관성 | 낮음 | 높음 |

## 7. 과거 버그 이력 및 수정 내용

### Bug 1 — 4배 중복 집계 (2026-05-18 수정)

**원인**: `소계/계` 행을 제외하지 않아 개별 행 합산에 공식 집계 행까지 포함됨.

```
수소/승용/비사업용: T
수소/승용/사업용:   T
수소/승용/계:       2T  ← 이 행도 합산 (중복)
수소/소계/비사업용: T
수소/소계/사업용:   T
수소/소계/계:       2T  ← 이 행도 합산 (중복)

실제 T 대신 코드는 8T를 합산
```

**수정**: `vehicle == "소계" or usage == "계"` 행을 제외하는 필터 추가.

### Bug 2 — 2018~2022 파싱 0건 (2026-05-18 수정)

**원인**: 병합 셀로 인해 B열이 None인 행을 `vehicle == ""` 조건으로 전부 건너뜀.

**수정**: B열 Forward-fill 구현 (`current_vehicle` 변수로 이전 값 유지).

### 최종 리팩토링 — 소계/계 직접 읽기 방식으로 전환 (2026-05-18)

**변경 전**: 비사업용 + 사업용 개별 행을 모두 합산하는 방식  
**변경 후**: 각 연료 섹션의 `소계/계` 행 1개를 직접 읽는 방식

두 방식은 수학적으로 동일한 결과를 내지만, 소계/계 직접 읽기 방식이  
코드가 단순하고 정부 공식 집계값을 그대로 사용한다는 점에서 더 명확하다.
